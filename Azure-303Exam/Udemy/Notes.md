<mark>Resource group </mark>is a new approach to *group* a collection of assets in logical *groups* for easy or even automatic provisioning, monitoring, and access control, and for more effective management of their costs

<mark>An Availability Set </mark>is a logical grouping capability for isolating VM resources from each other when they’re deployed. It is essentially your way of designing how available your cloud infrastructure is going to be in the event of unplanned or planned maintenance.
<mark>Availability Sets </mark>it will increase the availability of your Applications on the other hand the <mark>Availability Zones</mark> allows you to deploy your VM’s into different data centers within the same region.

<mark>Syslog </mark>is an event logging protocol that is common to Linux. Applications will send messages that may be stored on the local machine or delivered to a Syslog collector. When the Log Analytics agent for Linux is installed, it configures the local Syslog daemon to forward messages to the agent. The agent then sends the message to Azure Monitor where a corresponding record is created.
